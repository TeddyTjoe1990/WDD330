Week 10 Notes: Validating forms | Using Fetch

  • Validating Forms

    • Form validation is something you find on most websites where you either sign up for or create an account for. You will need to provide things such as emails or passwords. Form validation makes sure that the user inputs the right things in the right places. For example, if you are entering your email, you will always have a @ and .com in the email.

    Below is an input box that will ask you for an email. Try entering something that is not an email. There's also a password with some requirements to meet. Try different things and hit submit to see what happens.

    HTML5 provides built-in form validation. That way, there's no need to use any Javascript in order to validate a form, at least for some things. You can include attributes on form elements in your HTML to use validation. Some attributes are required, minlength and maxlength, min and max, type and pattern.

  • Using Fetch

    • Fetch usage is usually synonymous with API. Fetch itself if translated literally means to take. In this case, we are talking web design, the use of Fetch is usually always connected to the API. The Fetch API allows us to asynchronously request for a resource. Use the fetch() method to return a promise that resolves into a Response object. To get the actual data, you call one of the methods of the Response object e.g., text() or json() . These methods resolve into the actual data.
